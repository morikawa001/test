臨床研究研修会 ダッシュボード

臨床研究研修会のマスターCSV管理、開催情報入力、日程自動計算、保存前確認、 更新CSV出力、テンプレート一括差し込みを行うためのWebアプリです。

1. 概要

本アプリは、臨床研究研修会の開催管理を1つの画面で行うための業務支援ツールです。 マスターCSVの読込、新規データ作成、既存Noの更新、開催関連日付の自動算出、確認画面での検証、 更新CSVのダウンロード、および docx / pptx / xlsx テンプレートへの一括差し込みに対応しています。

アプリ名 臨床研究研修会 ダッシュボード
主用途 研修会マスターの登録・更新・確認・出力
対応入力 マスターCSV、フォーム入力、テンプレートファイル
対応出力 更新CSV、差し込み済み docx / pptx / xlsx

2. 画面構成

画面は、操作系、進捗確認、入力フォーム、スケジュール表示、保存確認、出力結果、差し込み処理の各エリアで構成されています。 主要構成は Quick Operation、Annual Plan Parameter、Deadline Alert、Entry Console、Computed Schedule、 Master Table、Commit Check、Export Result、Template Injection です。

エリア名 役割
Quick Operation テーマ切替、マスターCSV読込、新規DB作成、確認画面への移動、No選択を行います。
Annual Plan Parameter 年度件数、開催済み件数、直近30日件数、追加予定件数をリング表示します。
Deadline Alert 7日以内に期限が迫るタスクを優先表示します。
Entry Console 研修会の基本情報、Zoom情報、説明文などを入力・編集します。
Computed Schedule 自動計算された起案日、HP公開日、締切日、実施済みチェックを表示します。
Master Table 現在保持しているマスター行一覧を確認します。
Commit Check 保存前の差分や注意事項を確認します。
Export Result 更新CSVの出力状態を表示します。
Template Injection docx / pptx / xlsx テンプレートへの一括差し込みを実行します。

3. 基本操作

  1. 「csv選択」からマスターCSVを読み込むか、「新規DB作成」で空のデータベースを作成します。
  2. 既存データを編集する場合は、No選択欄から対象Noを選び、「選択Noを反映」を押します。
  3. 必要項目を Entry Console で入力または修正します。
  4. 右側の Computed Schedule で自動計算内容とチェック状態を確認します。
  5. 「確認画面へ進む」または保存操作を行い、Commit Check で内容を確認します。
  6. 問題がなければ保存し、「更新CSVをダウンロード」から最新CSVを取得します。
  7. 必要に応じてテンプレート差し込みを行い、資料ファイルをまとめて出力します。

4. CSV読込と新規DB作成

マスターCSVは「csv選択」ボタンから読み込みます。ファイル選択欄はブラウザ標準の 「選択されていません」表示を使わない構成に変更されており、選択後は専用表示欄にファイル名が反映されます。

CSVを読み込むと、No順にデータが整理され、一覧表示、統計表示、期限アラート、差し込み対象Noが更新されます。 新規DB作成では既定ヘッダーを持つ空のマスターが生成され、1件目から登録を始められます。

5. 入力項目

Entry Console では、No、年度、開催日、曜日、メール用開催日、氏名、タイトル、講師名、 講義開始、研修会開始、終了、事前打合せ、質疑応答締切、質疑応答時間、謝金、対象、開催方法、 共催、Zoom ID、パスコード、Zoom URL、HP URL、目的、導入文、趣旨説明、講師紹介などを扱います。

分類 主な項目
基本情報 No、年度、開催日、タイトル、講師名、名前
時刻情報 講義開始、研修会開始、終了、事前打合せ、質疑応答締切
開催設定 対象、対象詳細、開催方法、共催、共催3
配信情報 Zoom ID、パスコード、Zoom URL、HP URL
説明文 目的、進行表導入、趣旨説明、講師紹介

6. 自動計算

開催日や時刻の入力内容に応じて、曜日、メール用開催日、入室時刻、所要時間、各締切日、 起案1、HP公開、起案2、起案3などが自動計算されます。

  • 起案1は開催35日前です。
  • HP公開は開催28日前です。
  • 起案2は開催7日前です。
  • 起案3は開催翌営業日基準で計算されます。
  • 院外締切は開催8日前基準で算出されます。
  • 資料関連締切やメール用日付も自動反映されます。

また、対象区分や共催選択に応じて、対象詳細や共催関連の表示文言も自動補完されます。

7. 保存と確認画面

保存時は、入力中のNoを基準に、既存Noがある場合は上書き、ない場合は新規追加として処理されます。 同一Noが複数行として増えることはなく、マスター上では1つのNoにつき1行のみ保持されます。

Commit Check では、保存予定件数、要確認件数、状態、新規追加または上書きの判定、 入力不足や形式不正などの注意事項を確認できます。

  • No未入力
  • 開催日未入力
  • タイトル未入力
  • 講師名未入力
  • 開始・終了時刻未入力
  • 終了時刻が開始時刻以前
  • URL形式不正
  • 謝金が数値でない

8. CSV出力

保存後は「更新CSVをダウンロード」から最新状態のCSVを取得できます。 出力ファイル名は、開催日とタイトルをもとに作成される運用です。

CSVには、基本項目に加えて、起案1 / HP / 起案2 / 起案3 のチェック状態も含めて保存されます。

9. テンプレート差し込み

Template Injection では、docx / pptx / xlsx を複数まとめて読み込み、選択Noのデータで一括差し込みできます。 テンプレート未選択時は待機表示となり、対象Noとテンプレートが揃うと一括出力が可能になります。

差し込みは、CSVヘッダー名に加えて、DATE_1、DATE_2、TITLE、SPEAKER、NO、PASSCODE、ZOOM_URL、 WEBINER_ID などの別名キーにも対応する構成です。出力ファイル名は No とタイトルを含む形で生成されます。

10. テーマ切替

「表示テーマ切替」ボタンで、ダークテーマとライトテーマを切り替えられます。 ダークテーマはグリーン端末調、ライトテーマは Notion 風の明るい業務UIとして設計されています。

11. 注意事項

  • CSV更新後は、必ず最新の更新CSVをダウンロードして保管してください。
  • 同じNoで保存すると既存行の上書き更新になります。
  • テンプレート差し込みは docx / pptx / xlsx に対応しています。
  • URL項目は http または https 形式で入力してください。
  • 開催日や時刻を変更したときは、自動計算欄と確認画面を必ず確認してください。
